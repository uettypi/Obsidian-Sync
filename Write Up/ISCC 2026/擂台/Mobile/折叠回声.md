
## 一、对 APK 的初步探索

首先观察一下该 APK 的基础属性。由于其体积非常袖珍，可以推测这是一道经过精简、主攻算法逻辑的逆向题。

压缩包内部的核心文件主要包含：

```text
AndroidManifest.xml
classes.dex
assets/sleep_loop.webp
resources.arsc
META-INF/...
```

在这些文件里，尤为重要的几项如下：

- `classes.dex`：包含了应用的 Java 层执行逻辑。
- `assets/sleep_loop.webp`：伪装成一张图片，但内部实际上被植入了额外的隐蔽数据。
- `META-INF`：存放签名信息，会在后续的校验流程中用到。

## 二、提取 dex 中的字符串

通过对 `classes.dex` 进行明文字符串的提取，我们能迅速锁定以下几个重要文本：

```text
ISCC{this_is_only_the_first_echo}
The first echo still sounds convincing.
first echo accepted, but not the real one.
listen carefully
sleep_loop.webp
ECH0
fold-echo
vm-seed
TraceRecorder
FirstEchoCodec
EchoGate
```

从上述结果不难看出：

- `ISCC{this_is_only_the_first_echo}` 的格式非常像我们要找的 flag。
- 然而代码中还包含了一句 `first echo accepted, but not the real one.`。
- 这表明刚才那个 flag 只是出题人刻意布置的障眼法。

恰好呼应了题目描述中所说的“第一声很像答案”。

## 三、探究主界面的交互逻辑

对 `MainActivity` 进行逆向分析后发现，该界面的交互流程相对简单：

- 应用的标题命名为 `EchoFold`。
- 当用户对标题执行长按操作时，会弹出以下提示：

```text
The first echo still sounds convincing.
```

- 点击提交后，程序会优先触发 `FirstEchoCodec.check(input)` 函数。
- 倘若输入的刚好是那个伪造的 flag，界面便会回显：

```text
first echo accepted, but not the real one.
```

- 只有避开这个陷阱，才会走入后续真正的验证环节：

```java
EchoGate.verify(context, input)
```

综上所述，主干流程其实就是：

```text
先骗你一次，再真正校验一次
```

## 四、第一重伪装：虚假 flag 所在处

观察 `FirstEchoCodec.check()` 函数，其代码逻辑毫无掩饰：

```java
static check(String s) {
    return "ISCC{this_is_only_the_first_echo}".equals(s);
}
```

显而易见，这仅仅是一道简单的字符串比对，完全没有涉及复杂的运算，属于第一级迷惑手段。

由此可见：

```text
ISCC{this_is_only_the_first_echo}
```

纯粹是个用来误导人的鱼饵，绝非最终目标。

## 五、核心验证逻辑：EchoGate.verify()

整个应用最核心的部分都集中在 `EchoGate.verify()` 方法内。

其执行脉络大体可以拆解为如下几个阶段：

1. 验证用户输入的字符串是否符合 `ISCC{...}` 的基本格式要求。
2. 假如输入的内容等同于之前的那个假 flag，则直接判定为失败并返回 `false`。
3. 将大括号内部的具体字符串提取出来。
4. 动态构造出 4 组用于后续解密的核心参数。
5. 解析 `sleep_loop.webp` 图片，剥离出其中的隐写数据块。
6. 将提取到的数据还原成一个名为 `EFVM` 的自定义虚拟机结构。
7. 启动该虚拟机引擎，并输出一段密钥流（Key Stream）。
8. 将输出流与用户的输入做异或（XOR）比对，以判定是否正确。

刚才提到的 4 组核心参数分别是：

- `resourceKey(context)`：基于 `R.styleable.EchoFoldPulse` 资源推算得来。
- `dexDigest(context)`：当前 APK 压缩包内 `classes.dex` 文件的 SHA-256 哈希值。
- `certDigest(context)`：应用专属签名证书的 SHA-256 指纹。
- `TraceRecorder.snapshot()`：一段写死的 6 字节常量数据。

出题人在此处的意图十分明显：

- 强制将 flag 的计算与 APK 自身的特征进行强绑定。
- 杜绝了单纯通过硬编码提取常量的做法。
- 迫使文件的资源、DEX 以及签名信息都成为解密流程中不可或缺的一环。

## 六、解析 TraceRecorder 组件

`TraceRecorder` 类的代码非常简短：

```java
TRACE = [0x11, 0x23, 0x7a, 0x42, 0x51, 0x66]
```

换算出来即为：

```text
11 23 7a 42 51 66
```

其中的 `snapshot()` 方法也只是单纯地返回这 6 个字节的拷贝，未包含任何动态生成机制。

因此，它在整个加密体系里的角色相当于一串固定的 Salt（盐）。

## 七、推算 resourceKey

`resourceKey()` 的生成依赖于资源数组 `R.styleable.EchoFoldPulse`。

查看对应的 `R$styleable.smali` 文件，可以找到：

```text
EchoFoldPulse = [
  0x7f010000, 0x7f010001, ..., 0x7f01001f
]
```

即一段包含 32 个递增整数的数组。

开发者使用了如下循环逻辑，最终拼凑出长度为 24 字节的 `resourceKey`：

```java
for i in range(24):
    a = style[i % len(style)]
    b = style[(i + 1) % len(style)]
    key[i] = ((b - a) & 0xff) ^ (((a & 0x7f) << 1) & 0xff) ^ ((i * 13) & 0xff)
```

经过运算得出的 `resourceKey` 具体如下：

```text
01 0e 1f 20 3d 4a 43 54 79 66 97 98 85 b2 ab dc f1 fe cf d0 2d 3a 33 04
```

转换为连续的十六进制字符串格式即为：

```text
010e1f203d4a43547966979885b2abdcf1fecfd02d3a3304
```

## 八、计算 dexDigest 与 certDigest

### 1. dexDigest 获取方式

代码会在运行时读取自身的 `classes.dex` 文件，并计算出它的 SHA-256 散列值。

这一步骤起到了防篡改和校验运行环境的作用。

### 2. certDigest 获取方式

同时，程序还读取了应用的签名证书，针对证书的原始二进制字节流执行 SHA-256 哈希计算。

我们可以通过执行 `keytool -printcert -jarfile EchoFold.apk` 命令来查看相关证书详情，提取到的 SHA-256 值为：

```text
1B:62:BC:B4:8D:A1:83:8D:72:A1:BA:81:2F:0F:AA:77:
0F:57:99:8C:5A:E5:95:A2:7F:57:41:B1:25:83:E9:84
```

在代码内部实际运算时，会把所有的冒号剔除：

```text
1B62BCB48DA1838D72A1BA812F0FAA770F57998C5AE595A27F5741B12583E984
```

## 九、揭秘第二层暗示：sleep_loop.webp 中的秘密

整个 APK 包内仅提供了一个媒体资源文件：

```text
assets/sleep_loop.webp
```

然而该文件的体积仅有 282 字节，对于一张 WebP 图像而言极为反常。

检视其底层十六进制字节后，我们可以发现：

```text
RIFF .... WEBP .... ECH0 ...
```

这表明即便它在宏观上保留了 WebP 与 RIFF 的封装格式，但出题人其实在里面强行安插了一个自定义的区块（chunk）：

```text
ECH0
```

在逆向代码时，我们发现 `readEchoChunk()` 方法执行了如下动作：

1. 确认文件的 Magic Number 是否为 `RIFF`。
2. 确认文件类型是否被标识为 `WEBP`。
3. 逐个解析内部的数据块。
4. 寻觅标识符为 `ECH0` 的特定块。
5. 提取并返回该块所包含的完整数据。

这正是题目中所隐喻的“第二声很像线索”。

其潜台词是：真正的加密信息已被悄悄封装在这个伪装成“回声”的资源中。

## 十、利用 fold-echo 相关密钥解密 ECH0

在成功捕获 `ECH0` 块的内容后，程序并没有直接进入解析环节，而是先套用了一套流加密方式的异或处理：

```java
seed = sha256("fold-echo" + resourceKey + dexDigest + certDigest + trace)
plain = xorStream(echoChunk, seed)
```

这里面的 `xorStream()` 算法细节如下：

```text
第 0 块：sha256(seed + 00 00 00 00)
第 1 块：sha256(seed + 01 00 00 00)
第 2 块：sha256(seed + 02 00 00 00)
...
```

程序通过不断递增计数器生成每组 32 字节的哈希值，并将它们按顺序与密文数据进行异或。

完成解密后，可以观察到明文的头部内容为：

```text
45 46 56 4d
```

对应的 ASCII 字符即：

```text
EFVM
```

这个魔数（Magic Number）强烈暗示着作者自行设计了一套小型的虚拟机（VM）指令结构。

## 十一、剖析 EFVM 的内部构造

经过异或还原后的 `ECH0` 数据，遵循着以下布局：

```text
0x00-0x03  magic = "EFVM"
0x04       保留字节
0x05-0x06  prog_len
0x07-0x08  out_len
0x09...    program
后续       encrypted_target
```

在我们当前题目提取出的样本里：

- `prog_len` 的值为 `192`
- `out_len` 的值为 `35`

这意味着：

- 虚拟机的可执行代码段占用了 192 字节。
- 该 VM 运行结束后，将会产出一串 35 字节长度的输出数据。

注意，35 这一数值十分敏感，因为它完美契合了目标 flag 在大括号内部的字符数量。

## 十二、探究 runVm 方法的内在逻辑

`runVm(program, resourceKey, dexDigest, certDigest, trace, out_len)` 可以说是这道题最为关键的破局点。

其运作机制如下：

1. 采用 `sha256("vm-seed" + resourceKey + dexDigest + certDigest + trace)` 的结果，对 8 个 32 位宽的寄存器进行状态初始化。
2. 按照每 4 个字节为一组的跨度，对指令进行解析。
3. 首字节的末尾 3 个比特位（Bit）决定了当前指令的操作类型。
4. 余下的 3 个字节则分别指定了参与运算的寄存器编号。
5. 根据指令类型，运用位循环移位、异或、加法计算以及位/字节翻转等混淆手段，不断更新寄存器内的数据。
6. 当全部代码片段执行终了，程序将吐出要求长度（即 35 字节）的流数据。

在这里必须注意到一个核心突破点：

```text
runVm 的输出与“输入内容”无关，只与“目标长度”有关
```

事实上，用户提交的字符串只在收尾阶段被派上了用场：

```java
inputBytes XOR vmOutput == encryptedTarget
```

基于异或运算的可逆性，我们可以推导出：

```text
inputBytes = encryptedTarget XOR vmOutput
```

换句话说，这道题完全没有暴力破解的必要，选手只要原原本本地把环境依赖和 VM 的执行流程用脚本重构一遍，就能轻松倒推出正牌的 flag。

## 十三、梳理解题全流程

综合上述所有的分析，解题的具体步骤可概括为：

1. 对目标 APK 进行解压缩处理。
2. 导出其内部的 `classes.dex` 文件。
3. 算出该 DEX 文件的 SHA-256 哈希值。
4. 读出应用自身的签名证书并取得其 SHA-256 特征。
5. 利用 `R.styleable.EchoFoldPulse` 中的常数，反解出 `resourceKey` 参数。
6. 解析 `sleep_loop.webp` 文件，把 `ECH0` 数据块剥离出来。
7. 使用前置步骤搜集到的因子加上 `fold-echo` 关键字，解开流加密，取得 `EFVM` 核心结构。
8. 分别分离出虚拟机指令集（`program`）以及预设的密文（`encrypted_target`）。
9. 编写对应的脚本还原 `runVm()` 行为，得到最终的密钥流（`vmOutput`）。
10. 将密文与密钥流实施异或操作，即可获取 flag 的核心内容。

经过运算后能够得到字符串：

```text
f0lded_echo_is_a_state_not_a_string
```

加上外壳后，标准答案即为：

```text
ISCC{f0lded_echo_is_a_state_not_a_string}
```

## 十四、完整的 Python 自动化解题脚本

以下提供了一份使用 Python 编写的完整解答代码。在确保文件路径配置无误的情况下，运行该脚本即可直接计算出 flag。

```python
import hashlib
import pathlib
import struct

base = pathlib.Path(r"E:\桌面\附件\区域赛\echofold_unzip")
classes = (base / "classes.dex").read_bytes()
webp = (base / "assets" / "sleep_loop.webp").read_bytes()

trace = bytes([0x11, 0x23, 0x7A, 0x42, 0x51, 0x66])
style = [0x7F010000 + i for i in range(32)]

resource = bytes(
    (
        ((style[(i + 1) % len(style)] - style[i]) & 0xFF)
        ^ (((style[i] & 0x7F) << 1) & 0xFF)
        ^ ((i * 13) & 0xFF)
    )
    & 0xFF
    for i in range(0x18)
)

cert = bytes.fromhex(
    "1B62BCB48DA1838D72A1BA812F0FAA770F57998C5AE595A27F5741B12583E984"
)


def sha256(data: bytes) -> bytes:
    return hashlib.sha256(data).digest()


def rotl(x: int, n: int) -> int:
    x &= 0xFFFFFFFF
    n &= 31
    return ((x << n) | (x >> (32 - n))) & 0xFFFFFFFF


def rotr(x: int, n: int) -> int:
    x &= 0xFFFFFFFF
    n &= 31
    return ((x >> n) | (x << (32 - n))) & 0xFFFFFFFF


def reverse_bytes(x: int) -> int:
    return int.from_bytes((x & 0xFFFFFFFF).to_bytes(4, "little")[::-1], "little")


def reverse_bits(x: int) -> int:
    x &= 0xFFFFFFFF
    return int(f"{x:032b}"[::-1], 2)


def xor_stream(data: bytes, key: bytes) -> bytes:
    out = bytearray()
    ctr = 0
    while len(out) < len(data):
        block = sha256(key + ctr.to_bytes(4, "little"))
        base_idx = len(out)
        for i, b in enumerate(block):
            if base_idx + i >= len(data):
                break
            out.append(data[base_idx + i] ^ b)
        ctr += 1
    return bytes(out)


dex = sha256(classes)
seed = sha256(b"fold-echo" + resource + dex + cert + trace)

pos = webp.find(b"ECH0")
size = struct.unpack_from("<I", webp, pos + 4)[0]
chunk = webp[pos + 8 : pos + 8 + size]
plain = xor_stream(chunk, seed)

assert plain[:4] == b"EFVM"

prog_len = (plain[5] << 8) | plain[6]
out_len = (plain[7] << 8) | plain[8]
program = plain[9 : 9 + prog_len]
enc = plain[9 + prog_len : 9 + prog_len + out_len]

vm_seed = sha256(b"vm-seed" + resource + dex + cert + trace)
init_state = []
for i in range(8):
    v = int.from_bytes(vm_seed[i * 4 : (i + 1) * 4], "little")
    v ^= ((-0x61C88647 * i) & 0xFFFFFFFF)
    init_state.append(v & 0xFFFFFFFF)


def run_vm(program: bytes, a: bytes, b: bytes, c: bytes, d: bytes, out_len: int) -> bytes:
    s = init_state.copy()
    i = 0
    while i + 3 < len(program):
        op = program[i] & 0xFF
        ia = program[i + 1] & 7
        ib = program[i + 2] & 7
        ic = program[i + 3] & 7

        x = (
            (a[(i // 4) % len(a)] & 0xFF)
            ^ (b[(i // 2) % len(b)] & 0xFF)
            ^ (c[(i // 3) % len(c)] & 0xFF)
            ^ (d[(i // 4) % len(d)] & 0xFF)
        ) & 0xFF

        kind = op & 7

        if kind == 0:
            t = (s[ib] + x + i) & 0xFFFFFFFF
            t = rotl(t, (ic + x) & 31)
            s[ia] = (t ^ s[ia]) & 0xFFFFFFFF
        elif kind == 1:
            t = (s[ia] + s[ib] + ((x << 1) & 0xFFFFFFFF) + ic) & 0xFFFFFFFF
            s[ia] = rotl(t, (ib + ic + i) & 31)
        elif kind == 2:
            t = reverse_bytes(s[ic])
            t = (t ^ s[ib]) & 0xFFFFFFFF
            t = (t + x) & 0xFFFFFFFF
            s[ia] = (s[ia] ^ t) & 0xFFFFFFFF
        elif kind == 3:
            old = s[ia]
            s[ia] = (s[ib] + x) & 0xFFFFFFFF
            t = rotr((s[ic] + i) & 0xFFFFFFFF, x & 31)
            s[ib] = (t ^ old) & 0xFFFFFFFF
        elif kind == 4:
            t = rotr((x ^ s[ib] ^ i) & 0xFFFFFFFF, (ic + ia + 1) & 31)
            s[ia] = (s[ia] + t) & 0xFFFFFFFF
        elif kind == 5:
            t = (s[ib] + s[ic] + x) & 0xFFFFFFFF
            t = rotl(t, (i + ia + ib) & 31)
            s[ia] = (t ^ s[ia]) & 0xFFFFFFFF
        elif kind == 6:
            t = (x ^ s[ia] ^ s[ib]) & 0xFFFFFFFF
            t = (t + reverse_bits((s[ic] + i) & 0xFFFFFFFF)) & 0xFFFFFFFF
            s[ia] = t
        else:
            t = (x ^ s[ic] ^ s[ib]) & 0xFFFFFFFF
            t = (t + i) & 0xFFFFFFFF
            s[ia] = (s[ia] + t) & 0xFFFFFFFF

        i += 4

    out = bytearray()
    for j in range(out_len):
        v = s[j & 7] ^ rotl(s[(j + 3) & 7], j & 31)
        v ^= a[j % len(a)] & 0xFF
        v ^= b[j % len(b)] & 0xFF
        v ^= c[j % len(c)] & 0xFF
        v ^= d[j % len(d)] & 0xFF
        out.append(v & 0xFF)

    return bytes(out)


vm_out = run_vm(program, resource, dex, cert, trace, out_len)
flag_body = bytes(x ^ y for x, y in zip(enc, vm_out)).decode("ascii")
flag = f"ISCC{{{flag_body}}}"
print(flag)
```

执行后将在控制台输出：

```text
ISCC{f0lded_echo_is_a_state_not_a_string}
```
