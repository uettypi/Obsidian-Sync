
## 核心剖析与思路

根据题干信息，能够提炼出四个核心元素：

```text
天象
人言
地盘
时令
```

此外，题目还给出了十分关键的暗示：

```text
有人记位，有人记数。
有人记辞，有人只记点画。
有人抄正文，有人只抄边角。
有人信时刻，有人信节候。
```

这其实是在暗示我们，针对这四份不同的材料，不能使用单一的思维去解析。它们各自运用了不同的编码或隐写逻辑（例如分别按位置、数值、字词、边缘特征等规则）进行信息隐藏。

在初步探索文件时，我们可能会发现一些长得很像答案或线索的字符串，比如：

```text
flag{meta_lovers_die_first}
qian_kun_kan_li
dont_use_the_order
```

然而，这仅仅是出题人布下的障眼法（兔子洞），它们既不是真正的 Flag，也不能作为最终排序的依据。

破局的真正核心在于，需要分别对这四件旧物进行独立解码，从而各自提取出一段隐藏文本。

## 第一阶段：探寻四件旧物中的隐藏密钥

经过对四份独立材料的深入剖析与提取，我们最终能搜集到如下四个字符串片段：

```text
天象：qW2O0z
人言：5bnImG
地盘：4yFuUc
时令：4Y0nV9
```

随后，依照题目背景里提示的先后顺序将它们无缝组装起来：

```text
qW2O0z5bnImG4yFuUc4Y0nV9
```

我们将拼接后的完整字符串命名为：

```text
mid = qW2O0z5bnImG4yFuUc4Y0nV9
```

紧接着，利用 Python 对该结果进行 MD5 哈希运算：

```python
import hashlib

mid = b"qW2O0z5bnImG4yFuUc4Y0nV9"
print(hashlib.md5(mid).hexdigest())
```

执行脚本后可得：

```text
228a53da12c42a65235ca2ebefa826c1
```

由此，我们就成功获取了第一个压缩包 `窥得天机.zip` 的解压口令：

```text
228a53da12c42a65235ca2ebefa826c1
```

## 第二阶段：突破 `窥得天机.zip`

利用上一步算出的 MD5 值对目标压缩文件进行解密提取：

```bash
7z x 窥得天机.zip -p228a53da12c42a65235ca2ebefa826c1
```

完成解压操作后，会释放出 64 个文本（txt）文件。

在这个环节，千万不能忽略了该压缩包自带的注释信息（zip comment）。

我们可以通过脚本来提取其内部的注释内容：

```python
import zipfile

with zipfile.ZipFile("窥得天机.zip", "r") as z:
    print(z.comment.decode())
```

提取出的结果为：

```text
q7LxP0aR9Yv+u1kC3wFjI2sQmD4oJ6GfXr5hN8KpTtUVeBylHcSgZdWiEOAbMn/z
```

观察这串数据，其长度刚好是 64 位，并且包含的所有字符均属于标准的 Base64 字典体系：

```text
ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/
```

基于这个特征，我们完全可以将注释串里的每一个字符转换成它在 Base64 编码表里的索引位置。

举个例子来说明映射关系：

```text
A -> 0
B -> 1
...
a -> 26
...
0 -> 52
...
+ -> 62
/ -> 63
```

巧合的是，刚才解压得到的文本文件数量同样也是 64 个，这足以证实我们的猜想：

> 注释字符串里的第 i 个字符，其转换后的索引值代表我们需要在第 i 个 txt 文件中提取对应位置的字符。

用代码逻辑表示即为：

```python
index = base64_table.index(comment[i])
answer += txt_files[i][index]
```

依循此规律遍历并提取所有字符，就能拼凑出进入下一环节的密钥：

```text
UQC8TBH7FkJn7bmz7ikjttXfwjvsp99CHBWmH7+M9HxppE7W7a68mo/Ua4cfQUq2
```

我们将这段口令标记为：

```text
seven_pwd = UQC8TBH7FkJn7bmz7ikjttXfwjvsp99CHBWmH7+M9HxppE7W7a68mo/Ua4cfQUq2
```

## 第四阶段：AES-CBC 算法逆向还原

回顾先前的解题流程，我们手中目前掌握了两个极具价值的字符串变量：

```text
mid = qW2O0z5bnImG4yFuUc4Y0nV9
seven_pwd = UQC8TBH7FkJn7bmz7ikjttXfwjvsp99CHBWmH7+M9HxppE7W7a68mo/Ua4cfQUq2
```

鉴于该题目具有层层递进的密码学结构，可以考虑按如下规则构建 AES 的解密参数：

```text
AES key = 提取 mid 变量的前 16 字节
AES iv  = 提取 seven_pwd 变量的前 16 字节
```

具体取值如下：

```text
key = qW2O0z5bnImG4y
iv  = UQC8TBH7FkJn7bmz
```

完整的解密 Python 脚本实现如下：

```python
from base64 import b64decode
from Crypto.Cipher import AES
from Crypto.Util.Padding import unpad

mid = b"qW2O0z5bnImG4yFuUc4Y0nV9"
seven_pwd = b"UQC8TBH7FkJn7bmz7ikjttXfwjvsp99CHBWmH7+M9HxppE7W7a68mo/Ua4cfQUq2"

key = mid[:16]
iv = seven_pwd[:16]

ciphertext_b64 = "7jgRTVRxS/HSsI673SWW2yglLJbYZq295x7Eu6qCEvymWnnhkwyDg/8CduA2gQpF"
ciphertext = b64decode(ciphertext_b64)

cipher = AES.new(key, AES.MODE_CBC, iv)
plaintext = unpad(cipher.decrypt(ciphertext), 16)

print(plaintext.decode())
```

跑通脚本即可打印出最终明文：

```text
ISCC{cd1e6a3c-8348-4d9e-a26c-59c269004f84}
```

## 自动化全流程复现脚本

以下是集成了全部关卡的自动化通关脚本。在运行前，请确保脚本所在工作目录下包含如下依赖：

- `窥得天机.zip`
- `终局.7z`
- 预备好一个可以放置解压出的 64 个文本文档的输出目录

特别注意：由于脚本中调用了系统级的指令来处理压缩包，因此要求您的操作系统环境中已安装并配置好了 `7z` 命令行工具。

```python
import os
import re
import zipfile
import hashlib
import subprocess
from base64 import b64decode
from Crypto.Cipher import AES
from Crypto.Util.Padding import unpad

# 整合第一阶段解析出的四段碎块
part1 = "qW2O0z"
part2 = "5bnImG"
part3 = "4yFuUc"
part4 = "4Y0nV9"

mid = part1 + part2 + part3 + part4
zip_pwd = hashlib.md5(mid.encode()).hexdigest()

print("[+] mid:", mid)
print("[+] 窥得天机.zip password:", zip_pwd)

# 执行命令解压第一个包：窥得天机.zip
subprocess.run([
    "7z", "x", "窥得天机.zip", f"-p{zip_pwd}", "-y", "-oout_zip"
], check=False)

# 抓取并解析 ZIP 文件的元注释内容
with zipfile.ZipFile("窥得天机.zip", "r") as z:
    comment = z.comment.decode()

print("[+] zip comment:", comment)

base64_table = "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/"

# 收集并整理释放出来的 64 个文本文档
# 注：实际环境中可能需根据题目的具体派发附件微调排序逻辑。
txt_dir = "out_zip"
txt_files = []
for root, _, files in os.walk(txt_dir):
    for name in files:
        if name.lower().endswith(".txt"):
            txt_files.append(os.path.join(root, name))

# 设计排序函数：优先基于文件名里的数字进行升序；若无数字则按系统默认的字典序进行。
def sort_key(path):
    name = os.path.basename(path)
    nums = re.findall(r"\d+", name)
    return int(nums[0]) if nums else name

txt_files = sorted(txt_files, key=sort_key)

if len(txt_files) != 64:
    print(f"[!] txt 文件数量不是 64，而是 {len(txt_files)}，请检查解压目录/排序规则")

seven_pwd = ""
for i, ch in enumerate(comment):
    idx = base64_table.index(ch)
    with open(txt_files[i], "r", encoding="utf-8", errors="ignore") as f:
        data = f.read()
    seven_pwd += data[idx]

print("[+] 终局.7z password:", seven_pwd)

# 暴力解锁最终的压缩包：终局.7z
subprocess.run([
    "7z", "x", "终局.7z", f"-p{seven_pwd}", "-y", "-oout_final"
], check=False)

# 定位并读取 终局.txt 里面的密文数据
final_txt = None
for root, _, files in os.walk("out_final"):
    for name in files:
        if name == "终局.txt" or name.lower().endswith(".txt"):
            final_txt = os.path.join(root, name)
            break

if final_txt is None:
    raise FileNotFoundError("未找到 终局.txt")

with open(final_txt, "r", encoding="utf-8", errors="ignore") as f:
    ciphertext_b64 = f.read().strip()

# 执行终极操作：AES-CBC 对称解密
key = mid.encode()[:16]
iv = seven_pwd.encode()[:16]
ct = b64decode(ciphertext_b64)
pt = unpad(AES.new(key, AES.MODE_CBC, iv).decrypt(ct), 16)

print("[+] flag:", pt.decode())
```
