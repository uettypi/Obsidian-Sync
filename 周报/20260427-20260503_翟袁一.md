## 1.## VNCTF2023 电子木鱼
1. 这道题的后端是Rust，完全没学过，但好在其逻辑不算复杂，语法也比较通用，结合着AI审完了。
2. 其实这道题在明白了原理后就很简单了，没有什么需要列出来的步骤，核心就一点：int的溢出，具体原理如下：
3. Cost操作的cost值初始为10，而quantity值由POST请求控制，故只要控制`cost*quantity`的值超过int的上限2147483647，cost值就会溢出为一个接近int下限的极大的负值。
4. 然后，这个负值在经过
```rust
if GONGDE.get() < cost as i32 {

return web::Json(APIResult {

success: false,

message: "功德不足",

});

}
```
时，由于此时`GONGDE.get()`的值为0，大于该负值，故并不会触发"功德不足"，而是能顺利地进入下一阶段。
5. 最后，在经过
```rust
if cost != 0 {

GONGDE.set(GONGDE.get() - cost as i32);

}
```
时，由于cost的值是一个负值，0减去一个负值就负负得正了，故最终GONGDE的值将会是一个接近int上限的正值，大于1_000_000_000，此时访问主页面即可获得flag。
