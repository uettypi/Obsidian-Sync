3. Cost操作的cost值初始为10，而quantity的值由POST请求控制，故只要控制`cost*quantity`的值超过int的上限2147483647，cost值就会溢出为一个接近int下限的极大的负值。
4. 然后，在经过
```rust
if GONGDE.get() < cost as i32 {

return web::Json(APIResult {

success: false,

message: "功德不足",

});

}
```
时，由于此时`GONGDE.get()`的值为0，大于该负值，故并不会触发"功德不足"的返回，而是能够顺利地进入下一阶段。
5. 最后，在经过
```rust
if cost != 0 {

GONGDE.set(GONGDE.get() - cost as i32);

}
```
时，由于cost的值是一个负值，而0减去一个负值就负负得正了，故最终GONGDE的值将会是一个接近int上限的正值，大于1_000_000_000，故此时直接访问主页面即可获得flag。
