## 1.ISCC2026 校赛 Web3 夜班审计台
写WP时靶机已经下线了，所以这里没有实操的过程，只有解题思路。
1. 靶机主页面的SQL注入是没用的，无论如何都会显示“请求速度过快”。实际上真正有利用价值的信息在`main.js`这个文件中，打开后可以看到如下的内容：
```js
document.addEventListener("DOMContentLoaded", () => {  
window.__buildTrace = "/.git/HEAD";  
});
```
2. 进入其中所示的`/.git/HEAD`路径后，触发下载了一个名为`HEAD`的文件，打开后可以看到如下的内容：`ref: refs/heads/master`。此时，可以确定的是，这里存在一个Git源码泄露的漏洞。
3. 接下来，利用GitHack这个工具拉取靶机`/.git`路径下的Git源码。其中有一个名为`legacy_probe_stub.py`的文件，内容如下：
```python
# legacy_probe_stub.py

# compact handover note for the audit platform cut-over

  

DEFAULT_AUDITOR = ("auditor", "audit2025")

INTERNAL_DEV_SECRET = "ISCC_2026_JWT_DEBUG_KEY_#9527"

JWT_ACCEPTED = ["RS256", "HS256"]

  
  

def decode_ticket(token):

"""

current branch:

if header.alg == "RS256": verify with audit_rsa_pub.pem

elif header.alg == "HS256": verify with INTERNAL_DEV_SECRET

normal login still issues role=user

raise NotImplementedError

  
  

def handover():

class TinyMaze:

MAP = [

"#########",

"#..#....#",

"#..#.#..#",

"#....#..#",

"#########",

]

  

def __init__(self, start=(1, 1)):

self.pos = list(start)

  

def move(self, dx, dy):

x = self.pos[0] + dx

y = self.pos[1] + dy

if self.MAP[y][x] != "#":

self.pos = [x, y]

return tuple(self.pos)
```
4. 很明显，我们拿到了伪造JWT时必需的密钥——`ISCC_2026_JWT_DEBUG_KEY_#9527`，接下来，只需要将登录路由中JWT的role由默认的user改为auditor，再用这个密钥加密，即可得到伪造后的JWT。![image.png](https://img.uettypi.top/2026/05/20260508180203740.png)
5. 利用伪造的JWT，就可以以auditor的身份登录到一个审计后台，页面如下：![image.png](https://img.uettypi.top/2026/05/20260508181007813.png)
6. 这里有两个坑，一个是签名的密钥，另一个是待签名内容的格式。原先我一直以为这里用于签名的密钥就是前面用于伪造JWT的，以为只有待签名内容的格式是不知道的，所以一直在那尝试各种各样的格式...
7. 后来我才发现，Git源码中还有一个文件，其内容如下：
```python
# legacy_probe_stub.py

# old night-shift fallback verifier kept for rollback testing

  

SERVER_SECRET = "ISCC_SERVER_SECRET_REAL"

LOCAL_ONLY = ("127.0.0.1", "::1")

AUDIT_NODE = "core-storage-01"

TIME_WINDOW = 60

  
  

def verify_probe(node_id: str, ts: int, sign: str) -> bool:

"""

internal/audit fallback:

msg = f"{node_id}:{ts}"

expected = HMAC_SHA256_hex(SERVER_SECRET, msg)

abs(now-ts) <= 60

remote_addr in LOCAL_ONLY

"""

raise NotImplementedError

  
  

class PixelRunner:

def init(self):

self.energy = 3

self.score = 0

  

def tick(self, move: str):

if move in {"left", "right", "jump"}:

self.score += 1

self.energy = max(0, self.energy - 1)

return self.score, self.energy

  
  

def demo_loop(script):

game = PixelRunner()

for move in script:

game.tick(move)

return game.score
```
8. 这下看懂了，这里用于签名的密钥不是前面的那个，而是`ISCC_SERVER_SECRET_REAL`，待签名内容的格式是`{node_id}:{ts}`。好了，接下来就只需要用HMAC_SHA256签一下名，再在60s的窗口期内发送请求，即可在响应中得到flag。
